===========================================
CARD VISUAL PREFAB - COMPONENT HIERARCHY
===========================================

This is the exact structure for your CardVisual prefab.
Create this hierarchy in your Unity scene, then save as prefab.

-------------------------------------------
ROOT GAMEOBJECT
-------------------------------------------
Name: CardVisual
Type: RectTransform
Size: 150 x 200
Pivot: 0.5, 0.5
Components:
  - RectTransform
  - CanvasGroup (added automatically by CardVisual.cs)
  - CardVisual.cs ← Main script

-------------------------------------------
CHILD 1: Card Frame/Background
-------------------------------------------
Name: CardFrame
Type: Image (UI)
Parent: CardVisual
Size: 150 x 200 (fill parent)
Anchors: Stretch
Sprite: Your card frame sprite (use Fantasy RPG CardFrame_02_Bg.png)
Color: White (or customize)
Material: None
Raycast Target: Checked ← Important for clicking!

-------------------------------------------
CHILD 2: Card Back (Hidden When Face-Up)
-------------------------------------------
Name: CardBack
Type: Image (UI)
Parent: CardVisual
Size: 140 x 190 (slightly smaller than frame)
Position: Center
Sprite: Your card back design (e.g., decorative pattern)
Color: White
Raycast Target: Unchecked

-------------------------------------------
CHILD 3: Suit Icon
-------------------------------------------
Name: SuitIcon
Type: Image (UI)
Parent: CardVisual
Size: 64 x 64
Position: Center (0, 0) or offset as desired
Sprite: Leave empty (set at runtime by CardVisual.cs)
Color: Red (changed at runtime for black suits)
Preserve Aspect: Checked
Raycast Target: Unchecked

-------------------------------------------
CHILD 4: Rank Text
-------------------------------------------
Name: RankText
Type: TextMeshProUGUI
Parent: CardVisual
Size: 100 x 80
Position: Center (0, 0) or offset
Font: Your Fantasy RPG font
Font Size: 48
Alignment: Center, Middle
Text: "A" (placeholder, changes at runtime)
Color: Red (changed at runtime)
Raycast Target: Unchecked

Optional: Add two more TextMeshPro elements:
  - TopRank (small text, top-left corner)
  - BottomRank (small text, bottom-right corner, rotated 180°)

-------------------------------------------
CHILD 5: Highlight Glow
-------------------------------------------
Name: HighlightGlow
Type: Image (UI)
Parent: CardVisual
Size: 160 x 210 (slightly larger than frame)
Position: Center (0, 0)
Sprite: Soft glow sprite (or use default UI sprite with bloom)
Color: Yellow (1, 1, 0.5, 0.5) - semi-transparent
Material: UI/Default (or add glow shader)
Raycast Target: Unchecked
Active: Unchecked ← Disabled by default, enabled when held

-------------------------------------------
FINAL HIERARCHY VIEW
-------------------------------------------

CardVisual (RectTransform 150x200 + CardVisual.cs)
├── CardFrame (Image - always visible)
├── CardBack (Image - hidden when face-up)
├── SuitIcon (Image - ♥♦♣♠ set at runtime)
├── RankText (TextMeshPro - "A" "K" "2" etc.)
└── HighlightGlow (Image - shown when held/winning)

-------------------------------------------
CARDVISUAL.CS INSPECTOR SETUP
-------------------------------------------

In the CardVisual component inspector, assign these references:

Visual Components:
  Card Frame: → CardFrame (Image)
  Card Back: → CardBack (Image)
  Suit Icon: → SuitIcon (Image)
  Rank Text: → RankText (TextMeshProUGUI)
  Highlight Glow: → HighlightGlow (Image)
  Canvas Group: → Auto-assigned (on same GameObject)

Card Sprites (assign your sprites):
  Heart Sprite: → [Drag heart sprite here]
  Diamond Sprite: → [Drag diamond sprite here]
  Club Sprite: → [Drag club sprite here]
  Spade Sprite: → [Drag spade sprite here]
  Wild Card Sprite: → [Optional - for wild cards]

Colors:
  Red Suit Color: RGB(255, 0, 0) ← Default red
  Black Suit Color: RGB(0, 0, 0) ← Default black
  Wild Card Color: RGB(255, 214, 0) ← Gold
  Held Color: RGB(255, 255, 128) ← Yellow glow
  Winning Color: RGB(0, 255, 128) ← Green glow

-------------------------------------------
WHERE TO GET SPRITES
-------------------------------------------

Option A: Use your existing Fantasy RPG UI assets
Location: /Assets/Layer Lab/GUI Pro-FantasyRPG/ResourcesData/Sprites/Component/Frame/
Files:
  - CardFrame_01_Bg.png (card background)
  - CardFrame_02_Bg.png (card border)

Option B: Find free card sprites online
Recommended sites:
  - Kenney.nl (free game assets)
  - OpenGameArt.org
  - Itch.io (search "playing cards sprites")

Option C: Create your own
Photoshop/GIMP:
  - Card frame: 150x200 PNG with border
  - Suit icons: 64x64 PNG for ♥♦♣♠
  - Card back: 150x200 PNG with pattern

Suit icon Unicode:
  ♥ U+2665 (Heart)
  ♦ U+2666 (Diamond)
  ♣ U+2663 (Club)
  ♠ U+2660 (Spade)

Or use simple colored circles/shapes!

-------------------------------------------
QUICK TEST
-------------------------------------------

After creating the prefab:

1. Drag CardVisual prefab into scene
2. Select it in Hierarchy
3. In Inspector, look at CardVisual component
4. Click "Initialize" button (if you add one) or play scene
5. Card should be visible
6. Click card in Play mode → should toggle held state (yellow glow, move up)

-------------------------------------------
SAVE AS PREFAB
-------------------------------------------

Once hierarchy is complete:
1. Select root CardVisual GameObject
2. Drag into Project window
3. Save to: /Assets/RogueDeal/Prefabs/UI/CardVisual.prefab
4. Delete from scene (you only need the prefab)

The CardHandUI will spawn instances of this prefab at runtime!
